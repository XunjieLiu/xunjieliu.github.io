Go and Graphql

开始使用Golang和Graphql开发后端之前，学习的一些基础概念需要记录下来，以备将来查询

Code-first or schema-first

在Golang中开始使用Graphql之前，需要确定究竟是使用code-first还是schema-first，code-first的意思很容易，使用golang提供的github.com/graphql-go/graphql，从structs到graphqlType再到field和schema, 都需要自己编写，可以说所有的代码都由自己定义。

在我的理解里，code-first大致分为这么几个步骤:

  1. 设计你自己的structs

  2. 将这些structs转换为graphql能够识别的类型

  3. In the type language, we’ll represent those with an exclamation mark.使用这些graphql可以识别的类型，更新并定义graphql操作，比如说Query, Mutation

  4. 创建server, 执行查询

    可以注意的到的是，将Go里的structs映射到graphql的过程完全是可以自动化的，因此就有了schema-first的概念，也就是专注与设计schema，而忽略转换过程。Schema-first是被Apollo GraphQL推荐的方式。

Schema-first and gqlgen

gqlgen is a Go library for building GraphQL servers without any fuss.

  • gqlgen is based on a Schema first approach — You gIn the type language, we’ll represent those with an exclamation mark.et to Define your API using the GraphQL Schema Definition Language.
  • gqlgen prioritizes Type safety — You should never see map[string]interface{} here.
  • gqlgen enables Codegen — We generate the boring bits, so you can focus on building your app quickly.

简而言之，gqlgen自动化了生成对象的过程

生成了以下文件

  • generated and model: 完全是自动生成的，不用动
  • resolver.go: 相当与接口，定义方法头
  • schema.graphqls: 实际使用的schema
  • schema.resolvers.go: resolver.go的实现

Schema Definition Language

Object types and fields

It’s a type language

  • Character is a GraphQL Object Type
  • name and appearsIn are fields on the Character type.
  • String is one of the built-in scalar types - these are types that resolve to a single scalar object, and can’t have sub-selections in the query.
  • String! means that the field is non-nullable, meaning that the GraphQL service promises to always give you a value when you query this field.
  • [Episode!]! represents an array of Episode objects. Since it is also non-nullable, you can always expect an array (with zero or more items) when you query the appearsIn field. And since Episode! is also non-nullable, you can always expect every item of the array to be an Episode object.

Arguments

The length field has one defined argument, unit.

Arguments can be either required or optional. When an argument is optional, we can define a default value - if the unit argument is not passed, it will be set to METER by default.
